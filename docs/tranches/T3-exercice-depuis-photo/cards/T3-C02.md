# T3-C02 — Capturer ou téléverser une image

## 1. Résultat attendu

Fournir un composant client accessible qui accepte une seule image JPEG, PNG ou WebP par fichier ou capture caméra, effectue les contrôles locaux préliminaires et affiche un aperçu en mémoire avant envoi.

## 2. Valeur visible pour l'utilisateur ou le jury

L'élève peut photographier sa feuille sur mobile ou choisir un fichier, voir exactement l'image sélectionnée, la remplacer ou l'annuler avant toute analyse.

## 3. Prérequis et dépendances

T3-C01 close et shell UI issu de T0 disponible. Dépendance directe : T3-C01. Le navigateur doit être en contexte sécurisé pour une expérience caméra fiable; l'upload fichier reste disponible lorsque `capture` est ignoré.

## 4. Périmètre inclus

Champ fichier `accept="image/jpeg,image/png,image/webp"`, indication `capture="environment"`, glisser-déposer si le shell le permet, aperçu via Object URL, états accessibles, limite locale de 10 MiB et bouton de suppression/remplacement.

## 5. Contrats, types, entrées et sorties

Entrée : un `File` unique. État client : `empty|selected|client_rejected|submitting`. Sortie vers T3-C03/C04 : `SelectedExerciseImage { file: File; previewUrl: string; declaredMime: "image/jpeg"|"image/png"|"image/webp"; byteLength:number }`. Le MIME client n'est qu'un signal UX et ne remplace jamais le décodage serveur. L'Object URL et le `File` restent exclusivement dans l'état mémoire du composant.

## 6. Séquence d'implémentation décidée

Sélectionner une image; vérifier présence, type déclaré et taille `1..10 MiB`; révoquer l'ancien Object URL; créer l'aperçu; permettre remplacer/annuler; émettre le `SelectedExerciseImage` validé seulement sur action explicite « Analyze »; révoquer l'URL sur remplacement, annulation et unmount. La création du multipart reste à la frontière serveur de T3-C03/C04 et n'est pas anticipée ici.

## 7. Cas d'erreur et comportements de repli

Fichier vide, multiple, supérieur à 10 MiB ou MIME non autorisé : message local actionnable et aucun envoi. HEIC/HEIF, GIF, SVG et PDF : refus explicite avec invitation à fournir JPEG/PNG/WebP. Capture caméra indisponible : le sélecteur de fichier natif reste le repli. Une extension correcte avec contenu corrompu peut franchir ce contrôle mais sera rejetée autoritativement en T3-C03.

## 8. Critères d'acceptation Given/When/Then

- Given un JPEG de moins de 10 MiB, When il est sélectionné, Then l'aperçu, le nom et l'action Analyze sont disponibles.
- Given un HEIC ou un fichier surdimensionné, When il est sélectionné, Then aucun multipart n'est envoyé et le format attendu est annoncé.
- Given une image déjà sélectionnée, When elle est remplacée ou le composant démonté, Then l'ancienne Object URL est révoquée.
- Given un navigateur qui ignore `capture`, When l'utilisateur active le contrôle, Then le choix de fichier reste utilisable.

## 10. Preuves à consigner avant clôture

Les tests composant exécutés couvrent les trois MIME autorisés, le fichier vide,
HEIC, la taille `10 MiB + 1`, le multi-fichier, l'état `submitting` et l'absence
de callback Analyze pour toute sélection rejetée. L'espion Object URL observe
la révocation exacte au remplacement, au rejet d'un remplacement, à
l'annulation et au démontage. Le test de page exécuté confirme la présence de
l'uploader dans le shell et l'état initial désactivé d'Analyze.

Aucun appareil physique, sélecteur caméra natif, viewport mobile Playwright ou
smoke réseau n'a été testé pendant cette requalification. Ces limites restent à
couvrir par le gate final commun de C08 lorsqu'elles sont applicables.

## 12. Hors périmètre

HEIC, PDF, crop/rotation manuels, galerie persistante, upload direct vers OpenAI, extraction et mutation GeoGebra.

## 13. Risques et mitigations

MIME client falsifié : validation serveur par décodage. Fuite mémoire d'Object URL : révocation sur toutes les sorties. Variance de `capture` : upload fichier toujours disponible. Photo contenant des données personnelles : avertissement UX de ne photographier que l'énoncé utile.
