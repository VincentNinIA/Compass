# T21-C01 — PRD du harnais d'investigation GeoGebra

## 1. Résultat attendu

Un agent distinct peut prendre T22-C01 et implémenter le nouveau harnais sans
devoir décider la vision, les permissions, les contrats, le scénario Varignon,
la migration ou les gates.

## 2. Valeur visible pour l'utilisateur ou le jury

La cible produit explique clairement comment Compass transforme GeoGebra en
parcours construire, explorer, conjecturer et démontrer, sous contrôle du
professeur et avec preuves déterministes.

## 3. Prérequis et dépendances

T20-C01 close, audit du runtime public T18 et exemple Varignon fournis par le
porteur du projet.

## 4. Périmètre inclus

PRD v2, exigences FR-42 à FR-55, décisions durables, architecture cible,
roadmap T22, huit cartes d'implémentation, références et prochaine action.

## 5. Contrats, types, entrées et sorties

La sortie principale est
`docs/PRD_GEOGEBRA_INVESTIGATION_HARNESS.md`. Aucun contrat runtime n'est créé
dans cette carte et toute capacité T22 reste explicitement cible.

## 6. Séquence d'implémentation décidée

Auditer l'existant, spécifier le produit, fermer les actions et permissions,
décrire Varignon, définir la migration, écrire les cartes T22, synchroniser les
pilotes, exécuter les gates documentaires.

## 7. Cas d'erreur et comportements de repli

Une capacité non prouvée dans T18 est décrite comme écart. Une décision laissée
ouverte est résolue dans le PRD avant fermeture. Aucun code n'est ajouté pour
contourner une ambiguïté documentaire.

## 8. Critères d'acceptation Given/When/Then

Given le PRD, When un agent lit T22-C01, Then il connaît exactement son scope et
ses gates. Given la liste d'actions, When elle est auditée, Then chacune possède
autorité, entrée, préconditions, effet, sortie, preuve, cleanup et échec.

## 9. Tests et commandes de vérification

`pnpm test:docs:t0` et `git diff --check`.

## 10. Preuves à consigner avant clôture

PRD présent, neuf nouvelles cartes enregistrées et pilotes synchronisés.
`pnpm test:docs:t0` passe avec 80 cartes, IDs, dépendances, références et formats
valides. `git diff --check` passe sans erreur d'espace.

## 11. Documentation à consulter

Workflow Nin-IA, API GeoGebra, anciens contrats T1 à T6, runtime public T14 et
PRD v2 local.

## 12. Hors périmètre

Runtime, UI, tests applicatifs, déploiement, GitHub, Vercel et Devpost.

## 13. Risques et mitigations

Le risque principal est de mélanger cible et état livré. Chaque document nomme
T22 comme backlog et conserve T18/T20 comme référence déployée.
