# T24-C03 — Déployer et qualifier le candidat T22

## Statut

`active`

## Intention

Servir publiquement l'artefact T22 exact avec les protections de C02.

## Périmètre

Déploiement sur le projet Vercel existant, empreinte source/build/runtime,
smokes desktop/mobile, Varignon professeur–élève, headers, fallback et rollback.

## Critères d'acceptation

- L'alias stable sert l'empreinte candidate attendue.
- Le parcours public protégé passe sans erreur console ni fuite de secret.
- Le rollback documenté ramène à un candidat connu en cas d'échec.

## Vérification

Build propre, smoke HTTPS, trois parcours publics sans retry, accessibilité,
responsive, routes dynamiques et contrôle des headers.

## Hors périmètre

Soumission Devpost et pilote de classe.

## Contrat d'implémentation actif

- `95e9933` est la base protégée. Le candidat qualifiable est son descendant
  corrigeant le fond de checklist détecté par l'audit WCAG public; il doit être
  déployé depuis un worktree détaché propre sur `compass-geotutor-demo`.
- L'alias n'est qualifié qu'après état READY, inspection de l'identité et trois
  parcours HTTPS sans retry : refus anonyme, publication professeur Varignon et
  ouverture élève/atelier GeoGebra.
- Les contrôles couvrent desktop, mobile, console, responsive, headers, routes
  dynamiques, WAF, scan de fuite et rollback connu vers T18.
- Aucun secret, code d'accès ou cookie n'est écrit dans Git ni dans les preuves.
