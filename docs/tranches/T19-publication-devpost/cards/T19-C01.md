# T19-C01 — Publication GitHub et fiche Devpost

## 1. Résultat attendu

Le candidat T18 est accessible sur une branche GitHub révisable et la fiche
Devpost Education contient une présentation fidèle sans être soumise.

## 2. Valeur visible pour l'utilisateur ou le jury

Le porteur peut relire une candidature complète, partager le code source exact
et distinguer clairement le candidat local T18 de la démo Vercel T17.

## 3. Prérequis et dépendances

T18-C01 close; autorisation explicite de push GitHub et de mise à jour Devpost.

## 4. Périmètre inclus

Branche, commits, push, pull request brouillon, copie Devpost, technologies,
liens, état distant et documentation pilote.

## 5. Contrats, types, entrées et sorties

Le projet Devpost 1327494 reste `submission_draft`. Les liens et technologies
sont remplacés uniquement par les valeurs relues; aucun champ de vidéo inventé.

## 6. Séquence d'implémentation décidée

Contrôler Git/auth et scope; créer la branche; indexer le candidat borné;
vérifier et committer; pousser et ouvrir la PR; mettre à jour puis relire
Devpost; fermer les pilotes et pousser la preuve documentaire.

## 7. Cas d'erreur et comportements de repli

Une erreur Git ou Devpost laisse la carte active et ne déclenche ni merge, ni
soumission, ni déploiement. Un conflit distant est rapporté sans écrasement.

## 8. Critères d'acceptation Given/When/Then

Given le worktree T18, When la publication se termine, Then la PR expose le
candidat complet et Devpost reste brouillon. Given l'alias Vercel T17, When la
description est publiée, Then elle n'affirme pas que T18 y est déjà déployé.

## 9. Tests et commandes de vérification

Exécuter le validateur documentaire, les contrôles de diff indexé, le scan de
secrets, puis relire GitHub et Devpost après mutation.

## 10. Preuves à consigner avant clôture

URL de branche/PR, commits poussés, URL et état Devpost, scope indexé et résultat
du validateur documentaire.

## 11. Documentation à consulter

Pilotes Nin-IA, dossier `docs/DEVPOST_SUBMISSION.md`, dépôt GitHub et fiche
Devpost authentifiée.

## 12. Documentation à mettre à jour

CONTRACT, TODO_NEXT, ROADMAP, cette carte et checklist Devpost.

## 13. Hors périmètre explicite

Merge, déploiement Vercel, vidéo, feedback, licence, retours humains et
soumission finale Devpost.

## 14. Definition of Done

La branche et la PR sont publiques, la fiche Devpost est synchronisée et reste
un brouillon; les pilotes consignent les identifiants réels.
