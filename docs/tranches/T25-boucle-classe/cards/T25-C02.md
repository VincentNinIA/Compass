# T25-C02 — Classes et élèves pseudonymes

## Statut

`backlog`

## Intention

Donner au professeur un groupe pilotable sans construire un dossier scolaire.

## Périmètre

Identité professeur limitée, création/archivage de classe, code d'invitation
rotatif, roster de pseudonymes, révocation et parcours de jonction élève.

## Critères d'acceptation

- Un code valide rejoint exactement une classe sans révéler son roster.
- Collision, code expiré et révocation échouent sans création partielle.
- Le professeur peut retirer un alias et déclencher sa suppression prévue.

## Vérification

Tests d'accès, concurrence de jonction, rotation/révocation, EN/FR, clavier,
mobile et absence de données nominatives.

## Hors périmètre

Parents, comptes élève email, administration d'établissement et messagerie.
